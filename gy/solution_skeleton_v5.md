> **[SCQA]**

> **C**: 但若继续缺乏全资产管理工具，将面临错过股票、黄金等大行情的风险，在同业中被动；根因是5大系统性能力断层
> **Q**: 这5个根因是什���？为什么现在必须解决？

### 1.1 国元自营战略：从"利率债单一配置"到"全资产精细化管理"

**业务现实**（来自国元自营部门）：
- 近**1000亿**自营资金，目标低波动6%稳定收益
- 若仅投利率债，在股票、黄金等大行情中将**无法参与、无法获益**
- 缺乏全资产工具将导致年终考核**落后同业**——不是投资能力问题，是**工具和视野**问题
- 需要从方向性投资转向**相对价值投资**（利差策略、基差策略、新老券策略）
- 需要**多策略、多timeframe**捕捉投资和交易机会，抹平波动
- 需要通过系统**减少滑点、降低回撤、提升multi-leg交易能力**

**战略愿景**：
- 以组合管理为核心——不是以交易为核心，这是Aladdin级思维高度
- 全资产品种覆盖——不是固收单品种，而是真正的多资产配置
- 全公司自营组合统一管理——打破部门墙，全局寻找最优配置
- 丰富投资策略——量化+相对价值+多timeframe，提高收益确定性

### 1.2 五大能力断层：为什么1000亿资金"赚不到该赚的钱"

> **McKinsey 问题树：从"为什么1000亿自营资金面临错过行情、收益不稳的风险"倒推5个根因（MECE）**

```
"为什么近1000亿自营资金难以实现低波动6%的稳定收益？"
│
├─ 断层1【配不好】1000亿资金无法在全资产中找到最优配置
│   ├─ 若仅投利率债，将错过股票、黄金等大行情 → 缺乏全资产组合管理工具
│   ├─ 无法分析跨资产相关��，找不到低波动低相关的最优组合
│   ├─ 无法做大规模what-if试算，组合调整靠"拍脑袋"
│   └─ 无法穿透到投资经理级别评估每个人的贡献和能力
│
├─ 断层2【算不快】利差/基差/相对价值策略缺乏实时计算支撑
│   ├─ 风险计量T+1批量跑批，盘中出现机会无法实时响应
│   ├─ 定价依赖经验和Bloomberg/Calypso，无法自主快速定价
│   ├─ 利差/基差/新老券等相对价值分析缺乏系统化工具
│   └─ 多策略、多timeframe的信号捕捉和回测能力缺失
│
├─ 断层3【控不住】回撤和流动性风险无法实时管控
│   ├─ 人工监控为主，回撤已经发生才发现 → 损失已成事实
│   ├─ 流动性风险不可见，大额持仓的流动性冲击无法预估
│   ├─ 缺乏极端压力测试能力（如924央行干预债市类事件的预案）
│   ├─ 合规检查事后而非事前，违规交易无法提前拦截
│   └─ 缺乏跨资产、跨组合的统一风险视图
│
├─ 断层4【连不通】从研究到交易的全链路割裂
│   ├─ 投研发现的利差/基差机会无法直接驱动组合调整
│   ├─ 组合决策到交易指令手工传递，滑点损失严重
│   ├─ multi-leg交易（如利差交易同时买卖两只券）缺乏系统支持
│   ├─ IBOR缺失导致数据孤岛，前中后台各看各的数
│   └─ 各柜台（场内/场外/期货）割裂，无法一体化执行
│
└─ 断层5【追不上】头部券商和国际标杆差距持续拉大
    ├��� 华泰/平安已建成一体化FICC平台，能力代差明显
    ├─ Aladdin/SecDB定义了组合管理的能力天花板
    ├─ 信创替代窗口期紧迫（Calypso/Bloomberg/Murex依赖）
    └─ 不行动 = 差距加速扩大，人才和客户都会流失
```

### 1.3 三大驱动力决定"必须现在行动"

| 驱动力 | 具体压力 | 不行动的后果 |
|--------|---------|-------------|
| **考核驱动** | 缺乏工具→错过行情→考核被动→人才流失风险 | 同业差距持续拉大 |
| **监管驱动** | "十五五"明确数字化转型+信创替代 | 监管合规风险 |
| **竞争驱动** | 华泰/平安平台化已进入深水区 | 窗口期关闭，追赶成本翻倍 |

#### 4.4.11 全包成本核算引擎 [解决: 算不快] — "每笔交易的真实成本，一分不差"

> **核心问题**: 很多自营部门不知道一笔交易的"全包成本"（all-in cost），导致看似盈利的持仓实际亏损

**成本构成精确拆解（以一笔1亿债券回购杠杆交易为例）**：
```
买入成本:
  面值                     1亿
  + 券商佣金               ~1bp = 1万
  + CFETS交易费            ~0.1bp = 0.1万
  + 中债/上清结算费         ~0.05bp = 0.05万
  = 买入全包成本            100,001.15万

持有成本（假设回购融资持有90天）:
  回购融资利率              1.8% × 1亿 × 90/365 = 44.4万
  + 回购交易费              ~0.05bp × 回购金额 × 次数

卖出成本:
  同买入（佣金+交易费+结算费）= ~1.15万

全包成本:  买入 + 持有 + 卖出 ≈ 46.7万
→ 真实净收益 = 卖出价-买入价-46.7万
→ 如果价差只有40万 → 这笔交易实际亏损6.7万（但看买卖价差以为赚了40万）
```

**核心功能**：
- **实时全包P&L**: 每个持仓自动计算 = 市价收益 - 融资成本 - 佣金 - CFETS费 - 结算费
- **融资成本管理**: 回购利率实时跟踪，加权融资成本仪表盘，与市场利率对比
- **佣金分析**: 按券商/品种/金额分层统计，识别佣金优化空间
- **CFETS/结算费自动归集**: 按CFETS和中债/上清的费率表自动计算
- **杠杆成本预算**: 在下单前模拟"如果加杠杆，全包收益是多少"
- **盈亏归因含成本维度**: 绩效归因新增"交易成本拖累"和"融资成本拖累"项

**潜在价值**（1000亿AUM，500亿杠杆融资）：
- 发现隐性亏损持仓: 假设5%持仓成本核算偏差>5bp → 500-2000万/年
- 融资成本优化: 回购利率精细管理-10bps × 500亿 → **5000万/年**
- 更好的投资决策: 基于真实成本选券/加杠杆 → 间接Alpha贡献

```
优化后: 集中管理+最优利率选择，加权利率约1.7%
节省: 500亿 × 10bps = 5000万/年

具体优化手段:
- 回购到期日历统一管理 → 避免紧急续作的利率溢价
- 跨品种质押品优化 → 用最优质押品获取最低利率
- 回购期限结构优化 → 匹配持仓久期，减少利率错配
```

**精确成本核算详细计算**:
```
1000亿组合典型年度交易成本构成:
  佣金:      3000亿交易量 × 1bp平均佣金    = 3000万
  CFETS费:   3000亿 × 0.1bp               = 300万
  结算费:    3000亿 × 0.05bp              = 150万
  融资成本:  500亿杠杆 × 1.8%             = 9亿
  总成本:    约9.35亿/年

  如果这些成本的5%（4675万）因核算不精确而"隐藏"
  → 精确核算可以发现并优化 500-2000万/年 的隐性损失
```

**人效提升详细计算**:
```
自动化替代的人工工作:
  再平衡计算:   15 PM × 2h/周 = 1500h/年    → 自动化
  现金流管理:   1 FTE × 50% = 1000h/年      → 自动化
  多实体合并:   2 FTE × 25% = 1000h/年      → 自动化
  限额检查:    1 FTE × 30% = 600h/年       → 自动化
  报告制作:    3人 × 20% = 1200h/年        → 自动化
  总计:       ~5300h/年 ≈ 2.5 FTE
  按80万/FTE → 200万/年 人力直接节省
  + 错误率降低带来的隐性价值 → 200-450万/年
```
