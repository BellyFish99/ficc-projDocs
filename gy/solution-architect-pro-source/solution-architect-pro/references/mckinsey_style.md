# McKinsey Style & Logic Authority

This document defines the standards for executive-level communication, ensuring every document and slide is authoritative, logical, and value-driven.

## 1. The Pyramid Principle (SCQA)
Always structure narratives using the SCQA framework:
- **S (Situation)**: What is the stable baseline? (e.g., "Guoyuan manages ¥100B in self-trading assets.")
- **C (Complication)**: What changed or is failing? (e.g., "Manual processes and T+1 risk are strategic bottlenecks.")
- **Q (Question)**: What is the core challenge? (e.g., "How can we achieve stable 6% returns with real-time risk control?")
- **A (Answer)**: The proposed solution. (e.g., "Implement the Full-Asset POMS as the 'Business Intelligence Layer'.")

## 2. Action Titles ONLY
Titles must be complete sentences that state a conclusion. NEVER use "Status Report" or "Risk Management Overview."
- **Bad**: "Risk Management Overview"
- **Good**: "Real-time Risk Control is the Survival Baseline for a ¥100B Portfolio"
- **Bad**: "Solution Architecture"
- **Good**: "The POMS Platform Acts as the 'Auto-pilot' Brain for Existing Data Assets"

## 3. The "So What?" Insight Box (核心启示)
Every slide or major section must have a dedicated callout box titled "对[Client]的启示" or "核心启示."
- **Goal**: Explicitly state the business impact of the technical detail.
- **Example**: "Insight: Every 0.5bp slippage reduction results in ¥15M annual profit."

## 4. MECE Logic
Categorize all points to be Mutually Exclusive and Collectively Exhaustive.
- **Check**: Is there any overlap between these categories? Is anything missing?
- **Pattern**: When listing features, group them by "Capability Leaps" (e.g., 配得优, 算得快, 控得稳, 连得通).

## 5. Visual Authority in Text
- Use **Bold** for emphasis.
- Use `> Quotes` for governing thoughts or executive summaries.
- Use `| Tables |` for comparisons (As-Is vs. To-Be).
- Use `---` to separate major logical blocks.
