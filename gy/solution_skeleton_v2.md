## 1. 我们对国元自营业务战略的理解

> 目的：让CEO感到"他们比我自己人还懂我们"

### 1.1 国元自营战略核心要点
- 以稳定收益（低波动6%）为核心目标
- 全公司自营投资组合的统一管理
- 全资产品种覆盖

### 1.2 战略背后的五大核心挑战（问题树 — MECE分解）

> **McKinsey 问题树：从"为什么6%很难"倒推出5个根因**

```
"为什么自营组合难以实现低波动6%的稳定收益？"
│
├─ 挑战1【看不清】组合全景不可见
│   ├─ 各品种分散在不同系统，无法统一视图
│   ├─ 真实持仓与估值不实时，决策基于过时数据
│   └─ 无法穿透到投资经理级别评估贡献
│
├─ 挑战2【算不快】计算能力制约实时决策
│   ├─ 风险计量批量跑批（T+1），无法盘中响应
│   ├─ 定价依赖经验和外部系统（Bloomberg/Calypso）
│   └─ 大规模what-if试算无法并行执行
│
├─ 挑战3【管不住】风控被动滞后
│   ├─ 人工监控为主，缺乏事件驱动的主动预警
│   ├─ 流动性风险不可见
│   └─ 合规检查事后而非事前
│
├─ 挑战4【连不通】前中后台割裂
│   ├─ 投研结论无法直接驱动组合调整
│   ├─ 组合决策到交易指令手工传递
│   └─ IBOR缺失导致数据孤岛
│
└─ 挑战5【比不过】与行业领先差距拉大
    ├─ 头部券商（华泰/平安）已建成一体化FICC平台
    ├─ 国际标杆（Aladdin/SecDB）定义了能力天花板
    └─ 信创替代窗口期紧迫
```

### 1.3 挑战的紧迫性：为什么是现在？
- "十五五"监管明确要求数字化转型与信创替代
- 头部券商平台化已进入深水区，窗口期正在关闭
- 利率下行周期中，稳定收益更依赖精细化管理而非市场Beta

## 4. 解决方案总体设计

> 目的：这是全文的核心——每一个方案模块都必须回答一个明确的问题
> **McKinsey铁律：No problem, no solution. No solution without a problem.**

### 4.1 方案愿景与设计原则

**愿景**: 构建国元自营业务的"数字化中枢神经系统"

**六大设计原则**:
1. **组合为核心** — 一切围绕组合管理展开，而非围绕交易品种
2. **实时为基线** — 估值、风控、头寸全部实时化，消灭T+1
3. **事件驱动** — 被动查询→主动推送，CEP引擎贯穿全平台
4. **全资产统一** — 一套模型覆盖所有品种，而非每个品种一套系统
5. **开放可扩展** — 算力弹性伸缩，模块可插拔，API优先
6. **信创优先** — 全栈国产化，消除对Calypso/Bloomberg的依赖

### 4.4 核心模块详解

#### 4.4.1 IBOR — 投资簿记统一底座
- 全资产全市场全生命周期统一簿记
- 前中后台一体化实时头寸
- 打破数据孤岛，统一数据标准与模型
- 监管"穿透式"要求的关键基础设施
- 为AI训练、量化分析提供高质量数据基础

#### 4.4.2 组合管理引擎
- 多资产统一Position Model（股/债/衍生品/基金/回购/非标）
- 多层组合树（公司→部门→策略→子组合）
- 真实组合 + Shadow Portfolio虚拟组合并行
- 组合再平衡与what-if试算
- 穿透式实时持仓敞口分析（多维度：品种/久期/评级/行业）

#### 4.4.3 高性能计算引擎
- **定价引擎**: 覆盖权益/利率/商品/外汇/衍生品，支持自定义结构化产品
- **实时估值引擎**: 行情驱动流式估值，非批量
- **风险计量引擎**: VaR/CVaR/Greeks/久期/凸性/DV01/CS01/信用风险
- **分布式计算**: K8s弹性扩缩容，支持大规模并行

#### 4.4.4 事件驱动监控引擎
- CEP复杂事件处理（行情变动/交易成交/风控越限）
- 可配置规则引擎（监控规则+自动触发动作）
- 事件总线统一分发

#### 4.4.5 组合绩效归因
- Campisi模型：收入效应/国债效应/利差效应/交易效应/残差
- 利率债归因：久期判断/曲线形变/交易择时
- 信用债归因：信用利差/行业配置/个券选择
- 从"经验驱动"升级为"数据驱动"

#### 4.4.6 指令全生命周期管理
- 指令生成→审批→拆分→下达→执行→反馈闭环
- 组合再平衡自动生成交易指令
- 合规预检内嵌（事前自动检查）
- 支持批量操作、模糊识别、自动撮合

#### 4.4.7 现券交易系统
- 统一行情中心（外汇交易中心/货币中介/QTrade/交易所）
- 意向管理全流程（录入→发送→撮合→达成）
- 自动回价、自动撮合、轮巡策略
- xBond点击成交集成

#### 4.4.8 策略投资平台
- CEP流引擎 + 债券曲线模型
- 宏观因子库接入
- AI引擎集成
- 策略回测与模拟

#### 4.4.9 信用风险管理
- 全生命周期：投前评级→投中审批→投后预警
- 信用评估引擎 + 预警管理
- 投资池/债券池/黑名单管理
- 违约管理与指标监控

#### 4.4.10 FOF管理
- 基金资产信息获取/导入
- 组合风控报告自动生成
- 穿透式底层资产分析

### 4.5 技术架构与关键技术选型
- 微服务架构（高性能、松耦合、可扩展）
- 事件驱动架构（EDA）
- 分布式计算框架
- 信创技术栈（国产OS/DB/中间件）
- API-First设计
- 灵活二次开发能力（平台级/模块级/框架级）

### 4.6 非功能性需求保障
- 性能：毫秒级实时计算
- 高可用：99.99%可用性
- 安全：金融级安全标准
- 合规：满足"十五五"监管要求
- 信创：全栈国产化适配

## 6. 实施路径

> 目的：让CEO觉得风险可控、节奏合理、短期可见效

### 6.1 总体策略：从底向上，分步见效

**三大原则**:
- 先底座后应用 — IBOR先行，确保数据根基
- 先核心后扩展 — 组合管理+风险先行，再扩展交易/策略
- 先自营后做市 — 自营场景验证后再延伸至做市/客需

### 6.2 三期规划

```
Phase 1（0-6月）: 筑基期 — "看得清"
├─ IBOR统一数据底座上线
├─ 组合管理引擎MVP（真实组合+基础分析）
├─ 实时持仓敞口分析
├─ 定价平台基础功能
└─ 里程碑：全自营组合可在一个平台看到实时全景

Phase 2（6-12月）: 赋能期 — "算得快、管得住"
├─ 高性能计算引擎（风险计量+实时估值）
├─ 事件驱动监控引擎
├─ 虚拟组合与what-if试算
├─ 绩效归因系统
├─ 指令全生命周期管理
├─ 信用风险管理
└─ 里程碑：实时风控上线，指令自动化生成

Phase 3（12-18月）: 进化期 — "连得通、比得过"
├─ 策略投资平台（AI引擎集成）
├─ 四向集成全面打通
├─ 现券交易系统升级
├─ FOF管理
├─ 做市能力储备
└─ 里程碑：完整POMS平台上线，对标行业领先
```

### 6.3 快速验证：Phase 1 的 "灯塔效应"
