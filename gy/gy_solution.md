### 1.1 国元自营战略：从"利率债为主的配置交易"到"全资产精细化的，动态的组合管理"

**业务现实**（来自国元自营部门）：
- 近**千亿**自营资金，目标**{{target_yield}}**稳定收益
- 若仅投利率债，在股票、大宗商品，黄金等大行情中将**无法参与、无法获益**
- 缺乏全资产工具将导致年终考核**落后同业**——不是投资能力问题，是**系统建设·和视野**问题
- 需要从方向性投资转向**相对价值投资**（利差策略、基差策略、新老券策略）
- 需要**多策略、多timeframe**捕捉投资和交易机会，抹平波动
- 需要通过系统**减少滑点、降低回撤、提升multi-leg交易能力**

**战略愿景**：
- 以组合管理为核心——不是以交易为核心，这是海外Aladdin级思维高度
- 全资产品种覆盖——不是固收单品种，而是真正的多资产配置
- 全公司自营组合统一管理——打破部门墙，全局寻找最优配置
- 丰富投资策略——量化+相对价值+多timeframe，提高收益确定性

### 1.2 五大能力跃升目标：千亿资金实现稳定收益需要什么

│   ├─ 需要全资产组合管理工具，覆盖股票、黄金、债券，大宗商品，基金等全品种

│   ├─ 建立自主定价能力，摆脱对Bloomberg/Calypso的依赖

- **与国元的关联**：POMS的愿景本质上是"自营版Mini-Aladdin"，千亿规模完全值得这个投入

- **关键启示**：相对价值交易（利差/基差策略）的核心前提是统一、实时、精确的定价

#### 2.2.3 国际供应商格局 — 信创替代打开国产化窗口

### 4.4 平台系统架构：核心平台 × 专项赋能系统群

华锐POMS平台由**两个层次**的系统构成，形成"1＋16"的有机整体：

- **核心平台（4.4.1）**：全资产组合管理系统 — 直接承载国元核心诉求，解决"千亿资金如何在全资产中找到最优配置"的根本问题，亦是本方案重点阐述的部分
- **专项赋能系统群（4.4.2）**：十六大协同引擎 — 支撑组合管理系统高效运转的必要基础。没有高性能计算，组合优化无法实时完成；没有事件风控，敞口管理失去护盾；没有指令执行引擎，策略信号无法落地

##### 4.4.1.5 Campisi 2.0 — 固定收益绩效归因深度拆解

> **Action Title**: 债券组合的收益”从哪里来”一直是黑箱——Campisi 2.0将固定收益每一分钱收益精确拆解到利率平移/曲线形变/信用利差/个券选择，让投资决策从”感觉对”变成”数据证明对”

**Campisi 2.0归因框架**：

```
五大效应完整拆解:

① 收入效应 (Income Effect)
   = 持有期间票息收入 + 摊销
   → 衡量”不做任何决策，静态持有的基础收益”

② 国债效应 (Treasury Effect) — 利率决策归因
   ├─ 平移效应: 利率曲线整体上移/下移对久期的影响
   │   = -Modified Duration × ΔYield(parallel shift)
   ├─ 斜率效应: 曲线斜率变化（10Y-2Y利差）的影响
   │   = 各期限桶 Key Rate Duration × 斜率变化
   └─ 曲率效应: 曲线形状（蝴蝶）变化的影响
       = 凸性调整 + 曲率Key Rate Duration × 曲率变化

③ 利差效应 (Spread Effect) — 信用决策归因
   ├─ DTS贡献: Duration Times Spread变化 × 持仓DTS
   │   = 量化信用利差整体走阔/收窄对组合的影响
   ├─ 行业配置效应: 超/低配某行业相对基准的贡献
   └─ 个券选择效应: 个券利差变动超出行业平均的部分

④ 交易效应 (Trading Effect)
   = 实际交易时机带来的超额收益（或损耗）
   → 衡量”交易择时能力” — 是Alpha还是损耗

⑤ 残差 (Residual)
   = 实际收益 - 上述四项之和
   → 通常<5bp，过大表明数据或模型需检查
```

**实战应用示例**：

| 归因项 | 本月贡献(bp) | 解读 |
|--------|------------|------|
| 收入效应 | +18.5 | 票息稳定贡献，符合预期 |
| 利率平移 | -12.3 | 利率上行拖累久期持仓 |
| 曲线斜率 | +3.2 | 短端下行、陡曲线收益 |
| DTS信用利差 | +6.8 | 信用利差收窄，信用债受益 |
| 行业配置 | +2.1 | 超配城投，行情正确 |
| 个券选择 | +4.7 | 选券Alpha，高于行业平均 |
| 交易效应 | -1.2 | 换手时机略有损耗 |
| **合计** | **+21.8** | **与实际收益匹配** |

**核心价值**：投资经理第一次能清楚知道”这个月、这周、今天的超额收益，有多少是运气（市场给的），有多少是真本事（选券/择时Alpha）”——绩效考核从主观印象变成数据说话

##### 4.4.1.6 Brinson模型 — 资产组合绩效归因

> **Action Title**: 资产组合的收益来自”配了什么”还是”选了什么”，一直难以量化——Brinson模型将每一笔超额收益精确拆分为大类资产配置效应与个股/个基选择效应

**Brinson归因框架**：

```
三层归因结构:

第一层：大类资产配置效应 (Allocation Effect)
  = (组合权重 - 基准权重) × (基准资产收益 - 基准总收益)
  → 回答: “超配/低配某类资产的决策，贡献了多少收益？”
  示例: 超配黄金10%，黄金当月+5% vs 基准+2% → 配置贡献 +0.3%

第二层：个券/个基选择效应 (Selection Effect)
  = 基准权重 × (组合资产收益 - 基准资产收益)
  → 回答: “在同类资产里，选了更好的标的吗？”
  示例: 权益仓位内，选的股票+8% vs 沪深300 +5% → 选择贡献 +0.15%

第三层：交互效应 (Interaction Effect)
  = (组合权重 - 基准权重) × (组合资产收益 - 基准资产收益)
  → 超配了又选得好（或超配了但选得差）的综合效应

穿透分析能力:
  • 支持多层穿透: 资产大类→行业→子行业→个股/个基
  • 基金FOF穿透: 穿透底层持仓看真实行业/风格暴露
  • 时间序列: 每月/每季/年度累计归因趋势
```

**投资能力量化评估**：

```
核心结论输出（季度/周度/日度报告）:
  配置能力: 本季度 大类资产配置决策贡献 +X bp（高于基准）
  选券能力: 本季度 个股/个基选择贡献 +Y bp（高于同类基准）
  择时能力: 仓位调整时机贡献 +Z bp

与市场/同行对比:
  • 组合超额收益 vs 沪深300/中债综合/混合基准
  • 连续12个月滚动 — 识别能力稳定性 vs 运气因素
  • 投资经理排名（同部门横向对比，激励机制支撑）
```

**核心价值**：基金经理/投资经理的真实投资能力第一次可以被量化——绩效考核不再靠”感觉”，投委会可以用数据决定资源分配

##### 4.4.1.7 因子风险分解 + 边际风险贡献MCTR

> **Action Title**: 组合的VaR是1.2亿，但这个风险从哪里来？哪些持仓在消耗最多风险预算？——因子风险分解和MCTR回答这两个关键问题，让风险管理从”知道有多少风险”升级到”知道风险在哪里”

**因子风险分解**：

```
风险来源分解（全组合维度）:

系统性风险因子（可分散但需管理）:
  ① 利率风险: DV01敞口 × 利率波动率 → 贡献组合总风险 XX%
     细分: 短端(1Y内)/中端(1-5Y)/长端(5Y+)/超长端(10Y+)
  ② 信用风险: CS01敞口 × 信用利差波动率 → 贡献 XX%
     细分: AAA/AA+/AA城投/产业/金融各评级利差
  ③ 权益风险: Beta敞口 × 股市波动率 → 贡献 XX%
     细分: 大盘/小盘/价值/成长风格因子
  ④ 商品风险: 黄金/原油/有色价格敏感度 → 贡献 XX%
  ⑤ 汇率风险: 外汇敞口 × 汇率波动率 → 贡献 XX%

特异性风险（集中度风险）:
  = 个券/个股超出系统性因子解释的残余风险
  → 过高表明过度集中于单一发行人，需分散

输出形式:
  • 风险饼图: 各因子贡献占比可视化
  • 风险预算仪表盘: 已用/剩余/预警线
  • 历史趋势: 近10/30/60/90日因子风险结构变化
```

**边际风险贡献 (MCTR — Marginal Contribution to Risk)**：

```
MCTR计算逻辑:
  MCTR_i = ∂(组合波动率) / ∂(持仓_i的权重)
  → 表示”增加1%该持仓，组合总风险增加多少”

应用价值:
  ① 风险消耗排名: 找出”风险大户” — 哪10个持仓消耗了组合60%的风险预算
  ② 减仓优先级: 需要降风险时，先减MCTR最高的持仓，效率最大
  ③ 加仓引导: 想在风险预算内加仓，优先选MCTR为负（降低组合风险）的品种
  ④ 组合优化输入: 作为QP引擎的约束条件，实现”风险预算最优分配”

实时仪表盘输出:
  排名  持仓      市值(亿)  MCTR(bp)  风险贡献率  状态
   1   XX城投2030  5.2      18.3      12.1%      ⚠️ 过高
   2   XX转债      3.8      12.7       8.4%      正常
   3   XX股票      2.1      -3.2      -2.1%     ✅ 分散化贡献
```

**核心价值**：风险管理从”知道组合VaR是多少”升级到”知道每一笔持仓消耗了多少风险预算”——风险预算分配从粗放走向精细

##### 4.4.1.8 Component VaR + 主动风险归因

> **Action Title**: VaR是组合整体的风险数字，但管理者需要知道”哪类资产/哪个行业/哪个发行人在驱动这个VaR”——Component VaR和主动风险归因提供最细粒度的风险溯源

**Component VaR穿透分析**：

```
四层穿透结构:

第一层：资产类别
  债券VaR: X亿 (占比XX%)
  权益VaR: X亿 (占比XX%)
  商品VaR: X亿 (占比XX%)
  衍生品VaR: X亿 (对冲后净敞口)
  → 各类之间的分散化收益: -X亿（相关性<1带来的风险节省）

第二层：行业/子市场
  债券: 城投/金融债/产业债/国债各行业VaR
  权益: TMT/消费/金融/能源各行业VaR

第三层：评级/信用质量
  AAA: VaR X亿 (占比XX%)
  AA+: VaR X亿 (占比XX%)
  AA:  VaR X亿 (占比XX%)

第四层：单一发行人
  发行人A: Component VaR X亿 → 超过集中度限额 ⚠️
  发行人B: Component VaR X亿 → 正常范围 ✅
  发行人C: Component VaR X亿 → 提供分散化贡献 ✅

技术实现:
  • 历史模拟法: 基于500+日历史数据，99%置信度
  • Monte Carlo: 10,000次模拟，捕获非线性风险（期权等）
  • 参数法: 解析解，实时输出（毫秒级）
  • 三种方法互为验证，差异超过阈值自动预警
```

**主动风险归因（跟踪误差分解）**：

```
针对有基准组合的策略:

跟踪误差分解:
  总跟踪误差 = √(配置决策方差 + 选券决策方差 + 2×协方差)

  ① 配置效应跟踪误差
     = 大类资产偏离基准权重 × 各类资产波动率
     → 衡量”战术配置决策带来多少主动风险”

  ② 选券效应跟踪误差
     = 在各类资产内部选择标的 vs 基准成分券的差异
     → 衡量”选券能力带来多少主动风险”

信息比率 (IR) 监控:
  IR = 年化超额收益 / 年化跟踪误差
  目标: IR > 0.5（具备稳定选券能力的水平）
  预警: IR连续3月<0 → 策略有效性检验
```

**核心价值**：从”组合整体有多大风险”到”每一个持仓/行业/发行人对VaR的贡献是多少”，风险管理精细化到可操作层面

##### 4.4.1.9 敏感度分析 — 预判市场波动的”精密雷达”

> **Action Title**: 市场利率明天上行10bp，组合会亏多少？信用利差收窄20bp，能赚多少？——敏感度分析实时回答这些问题，让投资经理在市场波动发生前就看清组合的”风险地图”

**多维度敏感度指标实时仪表盘**：

```
利率敏感度:
  • DV01 (Dollar Value of 01): 利率每变动1bp，组合P&L变化（元）
    组合DV01: -XXX万 → 利率上行1bp亏XXX万
  • 关键期限点DV01（Key Rate Duration）:
    1Y: -XX万  2Y: -XX万  5Y: -XX万  7Y: -XX万  10Y: -XX万  30Y: -XX万
    → 识别组合在哪个期限点最脆弱
  • 凸性: 利率大幅变动时的非线性保护（正凸性=利率大涨时损失递减）

信用敏感度:
  • CS01 (Credit Spread 01): 信用利差每变动1bp，组合P&L变化
    按评级: AAA CS01/AA+ CS01/AA CS01
    按行业: 城投CS01/金融CS01/产业CS01
  • DTS (Duration Times Spread): 信用风险的更精确度量

权益敏感度:
  • Delta: 股票组合对指数（沪深300/中证500）每变动1%的敏感度
  • Beta: 系统性风险敞口，超过1则放大市场波动
  • 行业Beta: 各行业仓位对行业指数的敏感度

商品/汇率敏感度:
  • 黄金: 每盎司价格变动1美元，组合P&L变化
  • 原油: Brent原油每桶变动1美元的影响
  • 汇率: USD/CNY每变动100pips的外汇敞口影响
```

**冲击分析矩阵（Shock Grid）**：

```
利率冲击 × 信用利差冲击 组合P&L矩阵:

  信用利差           -50bp   -20bp    0bp   +20bp   +50bp
  利率      -50bp    +8500   +6200  +4100   +1800   -800
  利率      -20bp    +5200   +3100  +1200   -900   -3200
  利率        0bp    +4100   +1800    0    -2100   -5400
  利率      +20bp    +2800   +400  -1800   -4100   -7200
  利率      +50bp    +800   -1800  -4200   -6800   -9800

  → 红色区域：当前市场情景下最可能的损失区间
  → 投资经理一眼看出：最怕”利率大幅上行+信用利差走阔”的组合
```

**核心价值**：从”不知道市场变了组合会怎样”到”提前知道每种市场情景的P&L影响”——决策从被动应对变为主动预判

##### 4.4.1.10 情景分析与压力测试 — 历史重演 + 极端假设 + 反向推导

> **Action Title**: 2024年924事件证明：没有极端情景预案的自营部门，在极端市场中将面临灾难性损失——POMS三层压力测试体系，让”黑天鹅”不再是突然袭击

**三层压力测试体系**：

```
第一层：历史情景重演（基于真实事件）

  情景1: 2024年9月24日 — 央行货币政策转向
    触发: 央行意外宣布降息+降准，债券市场剧烈调整
    冲击参数:
      利率曲线: 短端-30bp / 中端-20bp / 长端-15bp（24h内）
      信用利差: 城投-10bp（风险偏好上升，权益暴涨）
      权益: 沪深300单日+9.6%
    → 组合今日损益: +XX亿（持有大量股票权益）或 -XX亿（纯债仓位）
    → 风险揭示: 如果仓位结构不同，同样的事件可能相差3-5亿

  情景2: 2019年5月 — 包商银行信用事件
    触发: 包商银行被接管，同业信用体系震动
    冲击参数:
      城投信用利差: AA+走阔+80bp / AA走阔+150bp
      同业存单: 中小银行利差+200bp
      流动性: 银行间质押回购紧张
    → 重现今日组合在该情景下的损失
    → 识别: 信用债集中度是否超过极端情景承受能力

  情景3: 2013年6月 — “钱荒”事件
    触发: 银行间流动性骤然枯竭，隔夜Shibor突破30%
    冲击参数:
      短端利率: +300bp（隔夜回购）
      长端利率: +100bp（恐慌性抛售）
      流动性: 大量持仓无法正常变现
    → 评估: 当前杠杆水平在极端流动性危机下的生存能力

第二层：假设情景模拟（自定义极端假设）

  投资经理可自由设定任意冲击组合，示例:
    “如果利率上行100bp + 信用利差走阔50bp + 股市下跌10%同时发生”
    “如果黄金下跌15% + 美元指数上升5%”
    “如果某个省份城投债出现违约，行业利差走阔200bp”

  系统即时输出:
    • 该情景下组合总损益: -XX亿
    • 最受损的前10个持仓排名
    • 流动性危机下能否正常偿还回购融资
    • 建议应急对冲方案（哪些衍生品可以对冲）

第三层：反向压力测试（倒推临界点）

  问题: “组合最多能承受多大损失？”（由风险限额决定）
  倒推: “什么样的市场情景会导致恰好触及止损线？”

  算法:
    ① 设定目标损失（如-2亿为止损线）
    ② 系统搜索最可能导致-2亿损失的利率/信用/权益冲击组合
    ③ 计算该情景发生的历史概率（如过去10年发生几次）
    ④ 输出: “您的临界情景是'利率上行65bp+信用走阔30bp'，历史上每3年发生1次”

  价值: 提前知道”最脆弱的临界点在哪里”，建立预案和止损机制
```

**核心价值**：2024年924事件教训——提前压测 = 有预案；没有压测 = 被动承受。POMS将极端情景从”事后复盘”变成”事前演练”，将损失从2亿压缩到0.8亿

##### 4.4.1.11 流动性风险管理 — 变现确定性量化

> **Action Title**: 大额持仓”卖不出去”的流动性风险，比价格风险更致命——POMS量化每个持仓的变现时间和市场冲击成本，让”能卖多快、要付多少代价”有数据支撑

**流动性风险量化体系**：

```
持仓变现能力评估（逐仓位）:

变现时间估算:
  算法: 结合持仓规模 / 日均成交量 / 市场深度
  假设: 每日变现不超过市场日均成交量的20%（避免自我冲击）

  典型示例:
  持仓              规模     日均成交量  估算变现天数  变现难度
  国债2034          10亿     50亿/日    1天          ✅ 极高流动性
  AA城投2028        5亿      3亿/日     8天          ⚠️ 中等流动性
  AA-产业债2030     2亿      0.5亿/日   20天+        🔴 低流动性
  非标ABS           3亿      无场内交易  协议转让      🔴 极低流动性

市场冲击成本估算:
  Almgren-Chriss模型: 冲击成本 ∝ √(变现规模/日均成交量)
  冲击成本 = 即时冲击 + 延迟冲击 + 市场反弹
  → 每个持仓变现时的”实际到手价格”与”当前市价”的差异
```

**流动性压力测试**：

```
场景: 需要在X天内变现Y亿资金（如满足赎回/回购到期）

系统分析:
  ① 可变现资产排序（流动性由高到低）
  ② 各持仓变现成本累计
  ③ 变现路径建议（先卖哪些，再卖哪些）
  ④ 最优变现方案的总成本 vs 被动全部卖出的成本差异

流动性调整VaR (LVaR):
  LVaR = 传统VaR + 流动性溢价风险
  → 在极端市场中，真实可承受损失 = VaR + 无法即时变现的成本
  → 比传统VaR更保守，更接近真实风险
```

**流动性监控仪表盘**：
- **流动性覆盖率**: 高流动性资产 / 未来30日现金需求（含回购到期）
- **流动性缓冲**: 可在1天/3天/5天/10天内变现的资产总额
- **回购到期日历**: 未来30日每日回购到期规模，对应可用资金

**核心价值**：在极端市场中，”能卖多少、能卖多快、要付多少代价”是生死问题——流动性风险量化让这三个问题有确定性答案

##### 4.4.1.12 衍生品工具矩阵 — 快速/灵活/稳健的组合对冲体系

> **Action Title**: 组合风险需要对冲时，投资经理需要的不是”我知道应该用衍生品”，而是”系统告诉我用哪个工具、用多少、怎么做”——衍生品工具矩阵将对冲从艺术变成科学

**衍生品工具覆盖矩阵**：

| 风险类型 | 对冲工具 | 用途 | 效率 |
|---------|---------|------|------|
| 利率风险 | 国债期货（2Y/5Y/10Y/30Y）| 快速调整组合久期 | 秒级，高流动性 |
| 利率风险 | 利率互换IRS（1Y-10Y）| 精确管理浮动/固定利率敞口 | 灵活，OTC |
| 信用风险 | 信用违约互换CDS | 对冲特定发行人违约风险 | 精准，OTC |
| 权益风险 | 股指期货（IF/IH/IC/IM）| 快速对冲Beta敞口 | 秒级，高流动性 |
| 权益风险 | 股指期权 | 非线性保护（尾部风险对冲）| 精准，成本可控 |
| 商品风险 | 黄金期货/黄金期权 | 对冲黄金价格风险 | 日内调整 |
| 汇率风险 | 外汇远期/掉期 | 锁定外汇敞口 | OTC，灵活 |

**对冲决策引擎**：

```
工作流:
  ① 输入: 当前组合风险敞口（来自DV01/CS01/Beta仪表盘）
  ② 目标: 设定对冲后的目标风险水平（如”将DV01从500万降到200万”）
  ③ 系统计算: 在成本最小的约束下，推荐最优对冲工具组合
     推荐: 卖出XX手10年期国债期货 + 做多XX笔5年IRS
     预期效果: DV01 500万 → 198万（-60.4%）
     对冲成本: XX万（含期货保证金占用+IRS溢价）
  ④ 一键生成对冲指令，流转执行管理引擎

多维对冲场景:
  “快速对冲”: 市场突变时，用国债期货1分钟内对冲100万DV01
  “精细对冲”: 用IRS精确管理特定期限的利率风险
  “尾部对冲”: 用期权对冲极端情景（购买虚值看跌期权）
  “跨资产对冲”: 股债联动行情下，同时用股指期货+国债期货组合对冲
```

**对冲效果追踪**：
- 对冲前/后组合风险指标对比
- 对冲成本 vs 风险降低收益（对冲效率评估）
- 对冲滑点追踪（执行价格 vs 决策价格）

**核心价值**：衍生品对冲从”知道要用但不知道怎么用多少”变成”系统给出最优对冲方案，一键执行”——将专业衍生品能力系统化、标准化

##### 4.4.2.8 全包成本核算引擎 [实现: 算得快] — "每笔交易的真实成本，一分不差"

> **核心问题**: 很多自营部门不知道一笔交易的"全包成本"（all-in cost），导致看似盈利的持仓实际亏损

**成本构成精确拆解（以一笔1亿债券回购杠杆交易为例）**：
```
买入成本:
  面值                     1亿
  + 券商佣金               ~1bp = 1万
  + CFETS交易费            ~0.1bp = 0.1万
  + 中债/上清结算费         ~0.05bp = 0.05万
  = 买入全包成本            100,001.15万

持有成本（假设回购融资持有90天）:
  回购融资利率              1.8% × 1亿 × 90/365 = 44.4万
  + 回购交易费              ~0.05bp × 回购金额 × 次数

卖出成本:
  同买入（佣金+交易费+结算费）= ~1.15万

全包成本:  买入 + 持有 + 卖出 ≈ 46.7万
→ 真实净收益 = 卖出价-买入价-46.7万
→ 如果价差只有40万 → 这笔交易实际亏损6.7万（但看买卖价差以为赚了40万）
```

**核心功能**：
- **实时全包P&L**: 每个持仓自动计算 = 市价收益 - 融资成本 - 佣金 - CFETS费 - 结算费
- **融资成本管理**: 回购利率实时跟踪，加权融资成本仪表盘，与市场利率对比
- **佣金分析**: 按券商/品种/金额分层统计，识别佣金优化空间
- **CFETS/结算费自动归集**: 按CFETS和中债/上清的费率表自动计算
- **杠杆成本预算**: 在下单前模拟"如果加杠杆，全包收益是多少"
- **盈亏归因含成本维度**: 绩效归因新增"交易成本拖累"和"融资成本拖累"项
- **押品智能化**: 智能押品篮管理，动态优化质押资产配置与置换路径，充分释放回购杠杆下的隐性融资价值

**潜在价值**（千亿AUM，500亿杠杆融资）：
- 发现隐性亏损持仓: 假设5%持仓成本核算偏差>5bp → 300-500万/年
- 融资成本优化: 回购利率精细管理-10bps × 500亿 → **2000-3000万/年**
- 更好的投资决策: 基于真实成本选券/加杠杆 → 间接Alpha贡献
